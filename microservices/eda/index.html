Application Details

This project is part of the research paper "The Migration of Monolithic Systems to AI-Based Microservices Using AI Design Patterns". The aim is to demonstrate how to migrate AI-based systems using design patterns. Due to the lack of available monolithic AI applications that also have a microservice-based version, we validated our approach by creating our own application.

This version of our AI system leverages an Event-Driven Architecture (EDA) for better scalability and flexibility. The application is an image classification system that uses the CIFAR-10 dataset to classify different images. In this setup, we have decomposed the ML pipeline into multiple microservices, each responsible for a distinct part of the workflow. The microservices include data loading and preprocessing, model training, model evaluation, and prediction.

  • Data Loading and Preprocessing Microservice: Handles the initial data preparation and communicates with the model training microservice via RabbitMQ to ensure a smooth flow of data.
  • Model Training Microservice: Processes the data, trains the model, and sends the trained model's path back through RabbitMQ to the evaluation microservice.
  • Model Evaluation Microservice: Assesses the performance of the trained model and sends the evaluation results to the prediction microservice.
  • Prediction Microservice: Uses the trained model to make predictions on new data.

To run this application, install the dependencies listed in the requirements.txt file and execute the following commands:

$ pip install -r requirements.txt
$ python flask_app.py

Architecture

Machine Learning Pipeline

You have two options for running the machine learning pipeline:

  • Run Each Step Separately: Execute the following steps individually:
    • Load and Preprocess Data: Prepare the data for training.
    • Train Model: Train the machine learning model using the preprocessed data.
    • Evaluate Model: Assess the performance of the trained model.
    • Predict: Use the trained model to make predictions on new data.
  • Run Full Pipeline: Execute all steps in one go with the "Run Full Pipeline" button.

All steps of the machine learning pipeline are performed on the same server.
